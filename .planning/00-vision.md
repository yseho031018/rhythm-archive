# Rhythm — Vision Document

> **한 줄 컨셉:** 나의 하루를 예술적인 리듬과 패턴으로 시각화하고, 스스로 깊이 있게 관찰할 수 있게 도와주는 디지털 자기관찰 아카이브 앱

---

## 1. 비전 (Vision)

**Rhythm**은 사용자가 매일의 감정, 에너지, 활동을 수동적으로 기록하는 대신, 자신만의 고유한 "리듬"을 시각적으로 발견하고 정리할 수 있게 돕는 앱이다.

Flutter CustomPainter와 Particle System을 활용한 유기적이고 예술적인 시각화(Multi-Layer 블렌딩)를 통해 데이터는 차갑은 숫자가 아닌, 살아 움직이는 형태로 표현된다. 통계 기반 패턴 분석(AI 없음)을 통해 사용자는 자신의 반복되는 주기와 상관관계를 객관적으로 인식하고, 이를 바탕으로 스스로 더 나은 결정을 내릴 수 있다.

Clean Architecture와 Riverpod을 기반으로 한 견고한 코드베이스는 장기적인 확장성과 유지보수성을 보장하며, Isar 로컬 DB는 오프라인 환경에서도 빠르고 안전한 데이터 관리를 가능하게 한다.

### 핵심 가치
- **시각적 자기관찰:** 데이터를 통해 '나'를 객관적으로 바라본다.
- **예술적 표현:** 기록은 아름다운 시각 언어로 재탄생한다.
- **주권적 인사이트:** AI가 아닌 사용자 스스로 패턴을 발견하고 해석한다.

## 2. 문제 정의 (Problem Definition)

### 2.1. 해결하고자 하는 문제

현재의 다이어리/기록 앱들은 대부분 다음과 같은 한계를 가지고 있다:

1. **기록의 피로도:** 매일 텍스트나 수치를 입력하는 것은 장기적으로 지속되기 어렵다. 입력 부담이 높을수록 이탈률이 증가한다.
2. **데이터의 비가시성:** 쌓인 기록들이 단순한 리스트나 표로 남아, 사용자가 자신의 패턴이나 흐름을 직관적으로 파악하기 어렵다.
3. **AI 의존의 수동성:** AI가 분석과 요약을 대신 해주는 앱들은 편리하지만, 사용자가 데이터를 직접 해석하고 성찰하는 능력을 약화시킬 수 있다.
4. **차갑은 UX:** 기능 중심의 UI는 사용자의 정서적 연결을 이끌어내지 못해, 앱에 대한 애착과 장기 사용을 저해한다.

### 2.2. 타겟 사용자

- 자기계발과 자기인식에 관심이 많은 20~30대
- 기록은 하고 싶지만 매일 쓰는 것이 부담스러운 사람
- 자신의 생활 패턴, 감정 주기를 시각적으로 파악하고 싶은 사람
- 예술적이고 감성적인 디자인을 선호하는 사람

### 2.3. 차별화 포인트

| 구분 | 기존 앱 | Rhythm |
|------|---------|--------|
| 입력 방식 | 텍스트 중심, 상세 기록 | 경량 입력 + 시각적 선택 |
| 데이터 표현 | 리스트, 표, 단순 그래프 | CustomParticle + Multi-Layer 블렌딩 |
| 인사이트 | AI 요약/추천 | 통계 기반 패턴 분석 + 사용자 주도 해석 |
| 감성 연결 | 기능적, 도구적 | 예술적, 몰입적 시각화 |

## 3. 성공 기준 (Success Criteria)

### 3.1. 정량적 기준 (Quantitative)

| 지표 | 목표값 | 측정 시점 |
|------|--------|-----------|
| 일일 활성 사용자 비율 (DAU/MAU) | 40% 이상 | 런칭 3개월 후 |
| 주간 평균 기록 횟수 | 5회 이상 | 런칭 3개월 후 |
| 월간 이탈률 (Churn Rate) | 15% 이하 | 런칭 3개월 후 |
| 앱 내 평균 체류 시간 | 3분 이상 | 런칭 3개월 후 |
| 주요 화면 렌더링 시간 | 16ms 이하 (60fps) | 개발 완료 시점 |
| 로컬 DB 쿼리 응답 시간 | 100ms 이하 | 개발 완료 시점 |

### 3.2. 정성적 기준 (Qualitative)

- 사용자는 자신의 "리듬"이라는 개념을 앱을 통해 자연스럽게 인식하고 언급할 수 있다.
- 기록이 의무가 아닌, 자신의 예술 작품을 감상하는 경험으로 느껴진다.
- 통계 분석 결과를 통해 사용자가 실제 생활에서 변화를 시도하고, 그 결과를 다시 기록하는 선순환이 형성된다.
- 오프라인 환경에서도 완벽하게 동작하여, 언제 어디서든 자유롭게 사용할 수 있다.

## 4. 핵심 기술 및 아키텍처 개요

- **Frontend:** Flutter (Dart)
- **State Management:** Riverpod
- **Architecture:** Clean Architecture (Presentation / Domain / Data)
- **Local Database:** Isar (고성능 NoSQL, 오프라인 우선)
- **Visualization:** CustomPainter + Particle System + Multi-Layer Blend Modes
- **Analytics:** 통계 기반 패턴 분석 (상관관계, 주기성 검출, 클러스터링) — AI/ML 미사용
