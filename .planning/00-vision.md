# 하루톡 비전과 문제 정의

## 한 문장 비전

> 긴 글을 쓰지 않아도 세 번의 선택으로 오늘을 남기고, 기록이 쌓일수록 나의 생활 패턴을 발견하게 한다.

## 문제 정의

기존 다이어리 앱은 사용자가 하루를 직접 떠올리고 긴 글로 정리해야 한다. 처음에는 기록 의지가 있어도 피곤한 날에는 작성 부담 때문에 쉽게 중단된다. 또한 기록이 쌓여도 목록이나 달력으로만 남아 “무엇을 한 날에 어떤 기분이었는지”를 찾기 어렵다.

## 공감 가능한 상황

- 과제를 마친 늦은 밤, 긴 일기를 쓸 힘은 없지만 오늘을 남기고 싶다.
- 기분이 좋거나 나빴던 이유를 정확히 설명하기 어렵다.
- 기록을 꾸준히 했지만 친구, 공부, 운동과 기분의 관계를 알고 싶다.

## 해결 아이디어

토리라는 캐릭터가 한 번에 질문 하나만 보여준다.

```text
오늘 기분은 어땠어?
→ 무엇과 함께한 하루였어?
→ 오늘 하루에 몇 점을 줄래?
→ 오늘의 한 줄 생성
```

사용자는 선택한 사실만 확인하고 필요하면 한 줄을 직접 수정한다. 기록은 한줄 목록, 감정잔디, 생활 패턴으로 다시 활용된다.

## 핵심 가치

| 가치 | 구현 |
|---|---|
| 기록 시작이 쉬움 | 한 화면에 질문 하나, 선택 중심 |
| 사용자가 통제함 | 다시 생성, 직접 수정, 삭제 |
| 오프라인 동작 | Drift/SQLite 로컬 DB |
| 기록이 쌓일수록 유용함 | 감정잔디, 통계, 생활 패턴 |
| 과도한 해석 방지 | 패턴 최소 표본과 만족도 차이 기준 |

## 성공 기준

- 사용자가 30초 안에 기록을 생성하고 저장할 수 있다.
- 새로고침 후에도 기록이 유지된다.
- 기록 목록과 감정잔디에서 저장 결과를 다시 확인할 수 있다.
- 누적 기록에서 키워드와 만족도 또는 기분 관계를 한 가지 이상 보여준다.
- 개발자가 구조, 기술 선택, 테스트, 빌드·배포를 말로 설명할 수 있다.
