## 1. 비전 (Vision)

**Rhythm의 비전:**  
> *모든 사람이 자신만의 삶의 리듬을 예술적으로 발견하고, 그 리듬 속에서 자신을 이해하며 더 나은 하루를 설계하는 세상을 만든다.*

우리는 하루하루를 살아가며 수많은 미세한 파장을 남깁니다. 기쁨의 파동, 지침의 잔류물, 무의식 속에서 반복되는 나만의 주기. 하지만 이 리듬들은 너무 작고 빨라, 우리는 그것들을 놓치고 살아갑니다.

Rhythm은 사용자의 하루를 예술적인 빛의 언어로 표현합니다. 감정과 에너지, 활동이 살아 움직이는 형태로 피어나고, 겹겹이 쌓여 하나의 풍경을 만듭니다. 차가운 숫자 대신, 따뜻하고 역동적인 시각 경험을 제공합니다.

Rhythm은 사용자를 대신해 정답을 내려주지 않습니다. 사용자가 스스로 반복되는 주기를 발견하고, 그 의미를 해석하며, 더 나은 하루를 설계하도록 돕습니다. Rhythm은 단순한 기록 도구가 아니라, 자신을 돌아보게 하는 거울이 되고자 합니다.

### 우리가 꿈꾸는 미래

> *수개월 후, 사용자가 자신의 캔버스를 바라볼 때 이렇게 말합니다.*  
> *"아, 나는 화요일마다 이런 색을 띠었구나. 운동을 한 날은 파장이 이렇게 달랐어."*  
> *그것이 단순한 통계가 아닌, 자기 자신에 대한 깊은 연민과 이해로 이어지는 순간.*  
> *그것이 Rhythm이 존재하는 이유입니다.*

### 핵심 가치

- **시각적 자기관찰 (Visual Introspection)**  
  데이터는 거울이 됩니다. 당신의 날들이 쌓여 만들어내는 풍경을 통해, '나'를 처음으로 객관적으로 바라보는 감각을 길러줍니다.

- **예술적 표현 (Artistic Expression)**  
  기록은 더 이상 의무가 아닙니다. 당신의 하루는 붓끝에서 입자가 되고, 빛이 되고, 하나의 예술 작품으로 재탄생합니다.

- **자기주도적 인사이트 (Self-Directed Insight)**  
  누군가가 정답을 알려주지 않습니다. 사용자가 직접 패턴을 발견하고, 해석하고, 그에 따라 행동하는 주체성을 회복합니다.

## 2. 문제 정의 (Problem Definition)

### 2.1. 우리가 마주한 공백

오늘날의 다이어리와 기록 앱들은 편리함을 약속하지만, 정작 우리에게 남기는 것은 공허함입니다.

1. **기록의 무게:** 매일 긴 문장과 복잡한 수치를 채우다 보면, 기록 자체가 또 하나의 할 일이 되어버립니다. 작은 마찰이 쌓여 결국 우리는 기록을 포기합니다.
2. **감춰진 흐름:** 쌓인 날들은 리스트와 표 속에 묻혀, 우리가 얼마나 아름답고 복잡한 존재인지를 직관적으로 느낄 수 없게 만듭니다.
3. **해석의 대리:** 자동화된 분석과 요약의 편리함 뒤에는, 스스로 생각하고 성찰하는 근육의 약화가 있습니다.
4. **정서의 단절:** 기능에만 충실한 차가운 인터페이스는, 우리가 앱에 애정을 느끼고 장기적으로 관계를 이어가기 어렵게 합니다.

### 2.2. 주요 사용자

- 자신을 더 깊이 이해하고 싶은, 자기계발에 관심 많은 20~30대
- 기록은 하고 싶지만, 매일 쓰는 것이 버거운 사람
- 자신만의 생활 주기와 감정의 파동을 시각적으로 만져보고 싶은 사람
- 기술이 아닌 예술과 감성으로 일상을 돌아보고 싶은 사람

### 2.3. 우리가 다르게 만드는 방법

| 구분 | 기존의 방식 | Rhythm의 방식 |
|------|-------------|---------------|
| 기록 | 텍스트 중심, 상세하고 무거움 | 가벼운 입력, 색과 터치로 빠르게 기록 |
| 표현 | 리스트, 표, 단순 그래프 | 입자가 춤추는, 살아있는 캔버스 |
| 인사이트 | 외부가 대신 요약하고 결론 내림 | 단서를 제공하고, 사용자가 직접 발견함 |
| 관계 | 기능적 도구 | 감성적 동반자, 몰입적 경험 |

## 3. 성공 기준

### 3.1. 프로젝트 완성 기준

| 기준 | 목표 |
|------|------|
| 기록 기능 | 사용자가 하루의 에너지, 감정, 활동을 입력하고 저장할 수 있다 |
| 시각화 기능 | 입력한 데이터가 Particle Canvas에 즉시 반영된다 |
| 히스토리 기능 | 저장된 기록을 날짜별로 다시 조회할 수 있다 |
| 패턴 확인 기능 | 1주 이상의 기록을 기준으로 간단한 통계 힌트를 보여준다 |
| 오프라인 동작 | 네트워크 없이도 입력, 저장, 조회가 가능하다 |

### 3.2. 발표/평가 기준

- 앱의 핵심 흐름을 직접 시연할 수 있다.
- 사용한 기술 스택과 선택 이유를 설명할 수 있다.
- Flutter 프로젝트 구조와 주요 디렉토리 역할을 설명할 수 있다.
- 로컬 DB에 데이터가 저장되고 조회되는 과정을 설명할 수 있다.
- 테스트 또는 수동 검증 결과를 문서로 남긴다.

## 4. 핵심 기술 및 아키텍처 개요

- **Frontend:** Flutter (Dart)
- **State Management:** Riverpod
- **Architecture:** Clean Architecture (Presentation / Domain / Data)
- **Local Database:** Isar (고성능 NoSQL, 오프라인 우선)
- **Visualization:** CustomPainter + Particle System + Multi-Layer Blend Modes
- **Analytics:** 통계 기반 패턴 분석 (상관관계, 주기성 검출, 클러스터링) — AI/ML 미사용
