## 1. 비전 (Vision)

**Rhythm의 비전:**  
> *모든 사람이 자신만의 고유한 삶의 리듬을 시각적으로 발견하고, 그 안에서 스스로를 깊이 이해하며 더 나은 하루를 설계할 수 있는 세상을 만든다.*

우리는 하루하루를 살아가며 수많은 미세한 파장을 남깁니다. 기쁨의 파동, 지침의 잔류물, 무의식 속에서 반복되는 나만의 주기. 하지만 이 리듬들은 너무 작고 빨라, 우리는 그것들을 놓치고 살아갑니다.

Rhythm은 당신의 하루를 예술적인 빛의 언어로 번역합니다. 당신의 감정과 에너지, 활동이 살아 움직이는 형태로 피어나고, 겹겹이 쌓여 하나의 풍경을 만듭니다. 차가운 숫자 대신, 따뜻하고 역동적인 시각의 시가 펼쳐지는 것입니다.

여기에는 누군가가 당신을 대신해 해석하지 않습니다. 당신 스스로 자신의 반복되는 주기를 발견하고, 그 의미를 부여하고, 더 나은 하루를 설계해 나갑니다. Rhythm은 도구가 아닌, 당신과 대화하는 거울이 되고자 합니다.

### 우리가 꿈꾸는 미래

> *수개월 후, 사용자가 자신의 캔버스를 바라볼 때 이렇게 말합니다.*  
> *"아, 나는 화요일마다 이런 색을 띠었구나. 울동을 한 날은 파장이 이렇게 달랐어."*  
> *그것이 단순한 통계가 아닌, 자기 자신에 대한 깊은 연민과 이해로 이어지는 순간.*  
> *그것이 Rhythm이 존재하는 이유입니다.*

### 핵심 가치

- **시각적 자기관찰 (Visual Introspection)**  
  데이터는 거울이 됩니다. 당신의 날들이 쌓여 만들어내는 풍경을 통해, '나'를 처음으로 객관적으로 바라보는 감각을 길러줍니다.

- **예술적 표현 (Artistic Expression)**  
  기록은 더 이상 의무가 아닙니다. 당신의 하루는 붓끝에서 입자가 되고, 빛이 되고, 하나의 예술 작품으로 재탄생합니다.

- **주권적 인사이트 (Sovereign Insight)**  
  누군가가 정답을 알려주지 않습니다. 당신 스스로 패턴을 발견하고, 해석하고, 그에 따라 행동하는 주체성을 회복합니다.

## 2. 문제 정의 (Problem Definition)

### 2.1. 우리가 마주한 공백

오늘날의 다이어리와 기록 앱들은 편리함을 약속하지만, 정작 우리에게 남기는 것은 공허함입니다.

1. **기록의 무게:** 매일 긴 문장과 복잡한 수치를 채우다 본, 기록 자체가 또 하나의 할 일이 되어버립니다. 작은 마찰이 쌓여 결국 우리는 기록을 포기합니다.
2. **감춰진 흐름:** 쌓인 날들은 리스트와 표 속에 묻혀, 우리가 얼마나 아름답고 복잡한 존재인지를 직관적으로 느낄 수 없게 만듭니다.
3. **해석의 대리:** 자동화된 분석과 요약의 편리함 뒤에는, 스스로 생각하고 성찰하는 근육의 약화가 있습니다.
4. **정서의 단절:** 기능에만 충실한 차가운 인터페이스는, 우리가 앱에 애정을 느끼고 장기적으로 관계를 이어가기 어렵게 합니다.

### 2.2. 우리가 함께하고 싶은 사람들

- 자신을 더 깊이 이해하고 싶은, 자기계발에 관심 많은 20~30대
- 기록은 하고 싶지만, 매일 쓰는 것이 버거운 사람
- 자신만의 생활 주기와 감정의 파동을 시각적으로 만져보고 싶은 사람
- 기술이 아닌 예술과 감성으로 일상을 돌아보고 싶은 사람

### 2.3. 우리가 다르게 만드는 방법

| 구분 | 기존의 방식 | Rhythm의 방식 |
|------|-------------|---------------|
| 기록 | 텍스트 중심, 상세하고 무거움 | 경량 입력, 색과 터치로 숨 쉬듯 기록 |
| 표현 | 리스트, 표, 단순 그래프 | 입자가 춤추는, 살아있는 캔버스 |
| 인사이트 | 외부가 대신 요약하고 결론 내림 | 단서를 주고, 당신이 직접 발견함 |
| 관계 | 기능적 도구 | 감성적 동반자, 몰입적 경험 |

## 3. 성공 기준 (Success Criteria)

### 3.1. 정량적 기준 (Quantitative)

| 지표 | 목표값 | 측정 시점 |
|------|--------|-----------|
| 일일 활성 사용자 비율 (DAU/MAU) | 40% 이상 | 런칭 3개월 후 |
| 주간 평균 기록 횟수 | 5회 이상 | 런칭 3개월 후 |
| 월간 이탈률 (Churn Rate) | 15% 이하 | 런칭 3개월 후 |
| 앱 내 평균 체류 시간 | 3분 이상 | 런칭 3개월 후 |
| 주요 화면 렌더링 시간 | 16ms 이하 (60fps) | 개발 완료 시점 |
| 로컬 DB 쿼리 응답 시간 | 100ms 이하 | 개발 완료 시점 |

### 3.2. 정성적 기준 (Qualitative)

- 사용자는 자신의 "리듬"이라는 개념을 일상 언어로 자연스럽게 사용하게 됩니다.
- 기록이 체크리스트가 아닌, 자신의 작품을 감상하는 일상의 의식이 됩니다.
- 분석 결과를 통해 사용자가 실제 삶에서 변화를 시도하고, 그 결과를 다시 기록하는 선순환이 만들어집니다.
- 오프라인의 고요한 순간에도, 앱은 완벽하게 동작하며 사용자의 자유로운 기록을 지킵니다.

## 4. 핵심 기술 및 아키텍처 개요

- **Frontend:** Flutter (Dart)
- **State Management:** Riverpod
- **Architecture:** Clean Architecture (Presentation / Domain / Data)
- **Local Database:** Isar (고성능 NoSQL, 오프라인 우선)
- **Visualization:** CustomPainter + Particle System + Multi-Layer Blend Modes
- **Analytics:** 통계 기반 패턴 분석 (상관관계, 주기성 검출, 클러스터링) — AI/ML 미사용
