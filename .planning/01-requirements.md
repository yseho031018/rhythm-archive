# 하루톡 요구사항

## 핵심 사용자 시나리오

### 시나리오 A — 오늘 기록

사용자는 기분, 키워드, 만족도를 선택하고 한 줄을 생성한다. 결과를 수정하거나 다시 생성한 뒤 저장한다.

### 시나리오 B — 과거 기록

사용자는 감정잔디의 빈 날짜를 선택해 해당 날짜의 기록을 작성한다.

### 시나리오 C — 생활 패턴

사용자는 통계 화면에서 반복된 키워드와 만족도·기분의 관계를 확인한다.

## MoSCoW

### Must

- 기분, 키워드, 만족도 순차 선택
- 규칙 기반 한 줄 생성
- 한 줄 수정, 다시 생성, 삭제
- 하루 한 개 기록과 과거 날짜 기록
- 로컬 저장과 재실행 후 복구
- 한줄 목록과 상세
- 감정잔디
- 주간·월간·연간 통계
- 생활 패턴 분석
- Flutter Web URL 시연
- 단위 테스트와 통합 위젯 테스트

### Should

- 사용자 키워드 추가·삭제·재사용
- 라이트·다크 테마
- 토리 상태별 이미지
- Pages 실패 시 로컬 서버와 영상 백업

### Could

- 실제 로컬 AI 서버
- Android APK
- 데이터 내보내기
- 알림

### Won't — 최종 평가 범위 제외

- 긴 일기 자동 작성
- 계정·클라우드 동기화
- 위치·날씨 자동 수집
- SNS 공유
- Isar 또는 서버 DB 전환

## 비기능 요구사항

| 구분 | 요구사항 |
|---|---|
| 사용성 | 질문은 한 번에 하나만 보여준다. |
| 안정성 | 외부 API 없이도 핵심 흐름이 동작한다. |
| 설명 가능성 | 사용자가 선택한 사실만 한 줄에 사용한다. |
| 성능 | 모바일 화면에서 전환과 스크롤이 자연스러워야 한다. |
| 품질 | analyze, test, Web Release 빌드가 통과해야 한다. |
| 배포 | 앱, 발표자료, WBS가 URL로 열려야 한다. |

## 완료 기준

- 30초 사용자 시나리오를 끊김 없이 시연한다.
- 자동 테스트가 모두 통과한다.
- README, AGENTS, setup, deploy, testing, architecture, ADR 문서가 현재 구현과 일치한다.
