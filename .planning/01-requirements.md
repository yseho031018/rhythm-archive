# Rhythm — Requirements Document

> 핵심 사용자 시나리오 3개 + MoSCoW 기능 분류

---

## 1. 핵심 사용자 시나리오 (User Scenarios)

### 시나리오 A: "오늘의 리듬을 남기는 다연"

**배경:** 다연(26세, 회사원)은 업무 스트레스로 인해 자신의 감정 상태를 파악하고 싶어한다. 매일 밤 잠들기 전, Rhythm을 연다.

**흐름:**
1. 홈 화면에서 오늘의 **에너지 레벨**(1~5)을 슬라이더로 입력한다.
2. **감정 키워드** 3개를 색상으로 된 터치 버튼에서 선택한다 (예: 피곤, 성취감, 불안).
3. **주요 활동**을 3개까지 태그 형태로 선택한다 (예: 업무, 운동, 독서).
4. 입력이 끝나면 홈 화면의 **Particle Canvas**가 실시간으로 업데이트되어 오늘의 리듬이 파동과 입자로 표현된다.
5. 캔버스를 길게 누륾면 오늘의 짧은 메모(선택 사항)를 추가할 수 있다.

**성공 기준:** 입력부터 시각화 확인까지 1분 이내에 완료된다.

---

### 시나리오 B: "나의 패턴을 발견하는 민수"

**배경:** 민수(29세, 프리랜서)는 2주간 꾸준히 기록한 후, 자신의 주간 패턴을 확인하고 싶어한다.

**흐름:**
1. 앱 하단의 **"패턴"** 탭으로 이동한다.
2. **주간 리듬 뷰**에서 7일간의 에너지 곡선과 감정 분포를 Particle 클러스터로 확인한다.
3. **통계 카드**에서 "수요일 에너지가 가장 낮음", "울동 후 성취감 87% 상승" 같은 상관관계 힌트를 본다.
4. 특정 날짜 범위를 선택하면 **Multi-Layer 블렌딩**으로 레이어별(감정/활동/에너지) 시각화를 겹쳐볼 수 있다.
5. 발견한 인사이트를 짧게 메모하여 **"나의 관찰 노트"**에 저장한다.

**성공 기준:** 2주 이상의 데이터가 축적되었을 때, 3개 이상의 개인적 패턴을 시각적으로 발견할 수 있다.

---

### 시나리오 C: "오프라인에서도 기록하는 지우"

**배경:** 지우(24세, 여행자)는 해외 여행 중 데이터 로밍이 불가능한 상황에서도 자신의 리듬을 기록하고 싶어한다.

**흐름:**
1. 비행기 모드 상태에서 앱을 실행한다.
2. 이전 기록들이 모두 로컬에 저장되어 있어 **Isar DB**에서 즉시 불러와진다.
3. 오늘의 리듬을 정상적으로 입력하고 저장한다.
4. 여행이 끝나고 Wi-Fi에 연결되면, 별도의 동기화 없이도 모든 데이터가 이미 로컬에 안전하게 존재한다.
5. (향후 확장 시) 백업 기능을 통해 클라우드로 난중 수출할 수 있다.

**성공 기준:** 네트워크 연결 없이 앱의 모든 핵심 기능(조회, 입력, 시각화, 통계)이 정상 동작한다.

## 2. 기능 분류 — MoSCoW

### 🔴 Must Have (없으면 안 되는 기능)

| ID | 기능 | 설명 | 연관 시나리오 |
|----|------|------|---------------|
| M1 | 일일 리듬 입력 | 에너지 레벨(1~5), 감정 키워드(다중 선택), 활동 태그(다중 선택) 입력 | A, C |
| M2 | Particle 기반 일일 시각화 | CustomPainter + Particle System으로 입력값을 실시간 시각화 | A |
| M3 | 주간/월간 리듬 히스토리 | 캘린더 또는 타임라인 형태로 과거 기록 조회 | B |
| M4 | 통계 기반 패턴 분석 | 상관관계, 평균, 빈도 등의 기초 통계로 패턴 힌트 제공 (AI 미사용) | B |
| M5 | 오프라인 완전 지원 | Isar 로컬 DB 기반, 네트워크 없이 모든 CRUD 동작 | C |
| M6 | Clean Architecture + Riverpod | 유지보수 가능한 레이어 분리 및 상태관리 | 전체 |

---

### 🟡 Should Have (있으면 훨씬 좋은 기능)

| ID | 기능 | 설명 | 연관 시나리오 |
|----|------|------|---------------|
| S1 | Multi-Layer 블렌딩 시각화 | 감정/활동/에너지 레이어를 겹쳐보는 고급 시각화 | B |
| S2 | 짧은 메모(노트) | 리듬 기록에 선택적 텍스트 메모 첨부 | A |
| S3 | "나의 관찰 노트" | 사용자가 패턴 분석 결과를 직접 메모하는 공간 | B |
| S4 | 일일 리마인드 알림 | 사용자가 설정한 시간에 기록을 유도하는 로컬 푸시 알림 | A |
| S5 | 데이터 익스포트 | JSON/CSV 형태로 로컬 데이터 백업 및 난중 수출 | C |

### 🟢 Could Have (여유 있을 때 추가)

| ID | 기능 | 설명 |
|----|------|------|
| C1 | 커스텀 테마/색상 | Particle 색상 팔레트 및 다크모드 커스터마이징 |
| C2 | 리듬 대결/공유 | 주간 리듬을 이미지로 저장하여 소셜 공유 |
| C3 | 월간 리포트 자동 생성 | 월마다 자동으로 요약된 리듬 리포트 페이지 생성 |
| C4 | 생체 리듬 연동 (수동) | 수면 시간, 기상 시간 등을 추가 입력 항목으로 확장 |

### ⚫ Won't Have (이 버전에서는 하지 않음)

| ID | 기능 | 이유 |
|----|------|------|
| W1 | AI 기반 감정 분석/추천 | 프로젝트 핵심 가치인 "사용자 주도적 인사이트 발견"과 배치됨 |
| W2 | 클라우드 동기화/백엔드 서버 | 로컬 우선 아키텍처의 정체성 유지, 복잡성 회피 |
| W3 | 외부 서비스 연동 (HealthKit, Google Fit 등) | 스코프 확대 방지, 1차 릴리즈에서 제외 |
| W4 | 멀티 유저/계정 관리 | 개인용 앱으로 스코프 한정 |
| W5 | 음성/사진 첨부 | 입력 경량화 원칙과 배치됨 |
