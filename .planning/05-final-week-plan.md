# 최종 평가 개발 계획

## 1. 계획 개요

- 계획 기간: 2026-06-10 ~ 2026-06-16
- 최종 평가 예정일: 2026-06-16
- 프로젝트 이름: 하루톡
- 프로젝트 방향: 선택 몇 번으로 오늘을 남기고, 쌓인 기록에서 생활 패턴을 발견하는 AI 한 줄 다이어리

### 핵심 가치

> 긴 일기를 쓰지 않아도 기분, 키워드, 만족도를 선택하면 AI가 오늘을 한 줄로 정리한다.

### 현재 진행 상태

- [x] 선택형 기록 화면과 AI 한 줄 더미 생성
- [x] 기록 목록·상세·감정잔디·월간 통계
- [x] SharedPreferences 로컬 저장
- [x] 한 줄 다시 생성·직접 수정·삭제
- [x] 같은 날짜 기록 교체 및 저장 실패 처리
- [x] 토리와 질문을 하나씩 진행하는 순차형 대화 UI
- [ ] 사용자 키워드와 과거 날짜 기록
- [ ] 생활 패턴 분석
- [ ] 실제 로컬 AI 연결과 배포 준비

### 기존 다이어리 앱과의 차이

- 일반 다이어리: 사용자가 직접 글을 작성하고 꾸민다.
- 선택형 기록 앱: 사용자가 고른 감정과 활동을 저장한다.
- 하루톡: 선택한 내용을 AI가 자연스러운 한 줄로 바꾸고, 누적 기록에서 감정과 활동의 관계를 알려준다.

## 2. 최종 사용자 흐름

```text
오늘의 기분 선택
→ 오늘의 키워드 선택
→ 만족도 선택
→ AI 한 줄 생성
→ 저장 또는 다시 생성·수정
→ 감정잔디와 기록 목록에 반영
→ 누적된 기록에서 생활 패턴 확인
```

AI는 긴 일기를 대신 작성하지 않는다. 사용자가 선택한 사실만 활용해 짧은 한 줄을 만들고, 기록을 해석하는 보조 역할만 담당한다.

## 3. 기능 우선순위

### Must: 최종 평가 전 반드시 완성

1. 로컬 저장
   - 앱을 다시 실행해도 생성한 기록 유지
   - 오늘 기록은 한 개만 유지

2. 한 줄 결과 제어
   - AI 한 줄 다시 생성
   - 사용자가 한 줄 직접 수정
   - 기록 삭제

3. 사용자 키워드
   - 직접 키워드 추가
   - 추가한 키워드 재사용

4. 과거 날짜 기록
   - 감정잔디의 비어 있는 날짜 선택
   - 선택한 날짜의 기록 작성

5. 생활 패턴 분석
   - 키워드별 평균 만족도
   - 키워드와 자주 함께 등장한 감정
   - 예: `친구를 만난 날은 평균 만족도가 1.2점 높아요.`

6. 평가 필수 산출물
   - Web Release 및 가능하면 APK
   - README, 배포 문서, 테스트 문서
   - 최종 발표자료, 3분 백업 영상, 예상 질문 답변

### Should: 시간이 허락하면 구현

- 기분·키워드·만족도별 기록 필터
- 로컬 AI 모델 서버를 이용한 실제 한 줄 생성
- AI 서버 연결 실패 시 규칙 기반 생성
- 기록 알림 설정

### Could: 평가 이후 후보

- 사진 첨부
- 사용자 키워드 아이콘·색상 설정
- 여러 종류의 AI 한 줄 문체
- 데이터 백업과 내보내기
- 앱 잠금

### Won't: 이번 평가 전 제외

- 긴 일기 생성
- 채팅형 AI 인터뷰
- 위치·날씨 자동 수집
- SNS 공유
- 사용자 계정과 클라우드 동기화
- 스티커 상점과 복잡한 꾸미기 기능
- 대규모 아키텍처·데이터베이스 전환

## 5. 화면별 최종 범위

### 기록

- 기록할 날짜 선택
- 기분 선택
- 기본 또는 사용자 키워드 선택
- 만족도 선택
- AI 한 줄 생성
- 한 줄 수정·다시 생성·저장

### 한줄

- 날짜, 기분, AI 한 줄 목록
- 기분·키워드 필터
- 상세 조회와 수정·삭제

### 감정잔디

- 날짜별 기분 색상
- 날짜 선택 시 한 줄 확인
- 비어 있는 과거 날짜 선택 시 기록 시작

### 마이

- 이번 달 감정 통계
- 연속 기록 일수
- 키워드별 평균 만족도
- 감정과 활동 관계를 설명하는 패턴 문장

## 6. AI 적용 원칙

- AI 입력은 날짜, 기분, 선택 키워드, 만족도로 제한한다.
- AI 출력은 한 문장으로 제한한다.
- 사용자가 선택하지 않은 사건이나 사실을 만들지 않는다.
- 의료 진단이나 감정 상태를 단정하지 않는다.
- AI 결과는 사용자가 수정하거나 다시 생성할 수 있다.
- AI 연결 실패 시 규칙 기반 한 줄 생성으로 자동 대체한다.

## 7. 중단 기준

- 06-10 종료까지 로컬 저장이 안정적이지 않으면 다른 기능보다 저장을 우선한다.
- 06-12 종료까지 패턴 분석이 불안정하면 가장 신뢰할 수 있는 분석 문장 한 종류만 남긴다.
- 06-13까지 실제 AI 서버 연결이 불안정하면 발표 시 규칙 기반 생성을 기본으로 사용한다.
- 06-13까지 APK 빌드가 해결되지 않으면 Flutter Web과 Windows Release를 배포 백업으로 사용한다.
- 06-14 이후에는 새로운 기능을 추가하지 않고 오류 수정과 발표 준비만 진행한다.
