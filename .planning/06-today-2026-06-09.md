## 권장 작업 순서

### 1단계. 프로젝트 방향 확정

예상 소요 시간: 30분

- 한 문장 가치 제안:

  > 몇 번의 짧은 대답만으로 AI가 오늘의 일기를 완성해주는 인터뷰 다이어리

- 핵심 사용자는 긴 일기를 꾸준히 작성하기 부담스러운 사람으로 설정한다.
- Wave Graph는 핵심 기능이 아닌 감정 결과 시각화로 사용한다.

완료 기준:

- [ ] 앱을 한 문장으로 설명할 수 있다.
- [ ] 기존 다이어리 앱과 다른 점을 한 문장으로 설명할 수 있다.

### 2단계. 핵심 사용자 시나리오 확정

예상 소요 시간: 45분

```text
1. 사용자가 오늘의 감정을 선택한다.
2. 사용자가 오늘 있었던 일을 짧게 입력한다.
3. 앱이 답변에 맞는 후속 질문을 한다.
4. 사용자가 한 번 더 답한다.
5. 앱이 대화를 바탕으로 짧은 일기 초안을 생성한다.
6. 사용자가 내용을 수정하고 저장한다.
7. 히스토리에서 저장된 일기를 확인한다.
```

완료 기준:

- [ ] 정상 흐름이 정리되어 있다.
- [ ] AI 서버가 연결되지 않은 실패 흐름이 정리되어 있다.
- [ ] 발표에서 보여줄 30초 시연 흐름이 정리되어 있다.

### 3단계. 화면과 데이터 구조 설계

예상 소요 시간: 1시간

필요 화면:

- 인터뷰 시작 화면
- 채팅형 인터뷰 화면
- 생성된 일기 수정 화면
- 저장된 일기 히스토리 화면

최소 데이터:

```text
DiaryEntry
- id
- date
- selectedEmotion
- interviewMessages
- generatedDiary
- editedDiary
- createdAt
```

AI 요청에 포함할 정보:

- 선택한 감정
- 현재까지의 질문과 답변
- 다음 질문 생성 또는 일기 생성 여부

완료 기준:

- [ ] 필요한 화면과 화면 전환이 정리되어 있다.
- [ ] 저장해야 할 최소 데이터가 정리되어 있다.
- [ ] AI에게 보낼 정보와 받을 결과가 정리되어 있다.

### 4단계. 기존 코드 재사용 범위 확인

예상 소요 시간: 1시간

재사용 후보:

- 감정 선택 UI
- 히스토리 탭
- 로컬 저장 구조
- Wave Graph
- 앱 테마와 하단 내비게이션

변경 후보:

- 오늘 탭을 AI 인터뷰 중심 화면으로 변경
- 짧은 메모를 인터뷰 답변으로 확장
- 패턴 탭을 이후 주간 요약 기능으로 활용

완료 기준:

- [ ] 유지할 코드와 변경할 코드가 구분되어 있다.
- [ ] 내일 구현할 첫 화면과 관련 파일이 정해져 있다.

### 5단계. 규칙 기반 대체 동작 설계

예상 소요 시간: 45분

AI 연결 실패 시에도 다음 흐름이 동작해야 한다.

| 감정 또는 답변 키워드 | 대체 후속 질문 예시 |
|---|---|
| 피곤, 지침 | 오늘 가장 에너지를 많이 사용한 일은 무엇이었나요? |
| 행복, 즐거움 | 오늘 가장 기억에 남는 순간은 무엇인가요? |
| 불안, 걱정 | 무엇이 가장 마음에 걸렸나요? |
| 성취, 뿌듯함 | 오늘 해낸 일 중 가장 만족스러운 것은 무엇인가요? |
| 기타 | 오늘을 가장 잘 보여주는 순간은 무엇이었나요? |

완료 기준:

- [ ] 최소 5개의 대체 질문 규칙이 있다.
- [ ] AI 없이도 일기 초안을 만드는 문장 조합 방식이 정리되어 있다.
