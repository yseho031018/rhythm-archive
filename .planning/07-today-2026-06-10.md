# 2026-06-10 오늘의 개발 일정

## 오늘의 목표

오늘은 새로운 기능을 많이 보여주는 날이 아니라, 현재 프로토타입을 **실제로 기록을 남길 수 있는 앱**으로 바꾸는 날이다.

현재 프로토타입의 가장 큰 문제는 생성한 기록이 앱을 다시 실행하면 사라지고, 생성 결과를 사용자가 수정하거나 삭제할 수 없다는 점이다.

오늘 종료 시점의 목표:

> 사용자가 AI 한 줄을 생성하고 수정·저장·삭제할 수 있으며, 앱을 다시 실행해도 기록이 유지된다.

## 오늘 반드시 완료

- [x] 프로토타입 기록 모델의 저장 형식 정의
- [x] SharedPreferences 기반 로컬 저장 연결
- [x] 앱 시작 시 저장된 기록 불러오기
- [x] AI 한 줄 다시 생성
- [x] AI 한 줄 직접 수정
- [x] 기록 삭제
- [x] 오늘 기록 재작성 시 기존 기록 교체
- [x] 저장·수정·삭제 테스트 추가
- [x] 모바일 화면에서 전체 흐름 확인

## 권장 작업 순서

### 1단계. 데이터 모델과 저장 구조 정리

예상 소요 시간: 45분

저장할 최소 데이터:

```text
DiaryEntry
- id
- date
- mood
- keywords
- satisfaction
- summary
```

결정 사항:

- 한 날짜에는 기록 한 개만 저장한다.
- 같은 날짜에 다시 기록하면 기존 기록을 교체한다.
- 사용자 키워드는 06-11 작업에서 별도로 저장한다.

완료 기준:

- [x] 기록을 JSON으로 변환하고 다시 복원할 수 있다.
- [x] 기존 더미 데이터와 실제 사용자 기록을 구분할 수 있다.

### 2단계. 로컬 저장 연결

예상 소요 시간: 1시간 30분

- `DiaryController`가 앱 시작 시 저장된 기록을 불러오도록 변경
- 생성·수정·삭제 후 자동 저장
- 저장된 기록이 없을 때만 데모 데이터를 보여주는 방식 결정
- 로딩 중 화면 상태 처리

완료 기준:

- [x] 기록 생성 후 앱을 다시 실행해도 유지된다.
- [x] 같은 날짜 기록이 중복되지 않는다.
- [x] 저장 실패가 앱 전체 오류로 이어지지 않는다.

### 3단계. 결과 화면에 사용자 제어 추가

예상 소요 시간: 1시간 30분

생성 결과 화면:

- `저장`
- `다시 생성`
- `직접 수정`

상세 화면:

- `한 줄 수정`
- `기록 삭제`

완료 기준:

- [x] 다시 생성할 때 같은 선택을 사용해 다른 문장이 나온다.
- [x] 사용자가 수정한 한 줄이 저장된다.
- [x] 삭제한 기록이 목록·감정잔디·통계에서 함께 사라진다.

### 4단계. 테스트와 모바일 검증

예상 소요 시간: 1시간

- JSON 직렬화 테스트
- 생성·수정·삭제 저장 테스트
- 같은 날짜 기록 교체 테스트
- `flutter analyze`
- `flutter test`
- 모바일 크기에서 결과 시트와 수정 화면 확인

완료 기준:

- [x] 분석과 테스트가 통과한다.
- [x] 긴 한 줄에서도 버튼과 텍스트가 겹치지 않는다.
- [x] 기록 화면부터 감정잔디 반영까지 흐름이 정상 동작한다.
