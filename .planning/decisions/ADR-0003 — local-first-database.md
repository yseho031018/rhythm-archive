# ADR-0003 — SharedPreferences 기반 로컬 우선 저장

- 상태: 채택
- 결정일: 2026-05-26
- 최종 검토일: 2026-06-13

## 상황

하루톡의 핵심 기록은 인터넷이 없어도 생성·조회되어야 한다. 초기 계획에서는 Isar를 검토했지만, 최종 평가 전에는 기록 수가 적고 복잡한 검색보다 안정적인 저장·복구와 설명 가능성이 중요했다.

## 결정

`DiaryRepository` 인터페이스 뒤에서 SharedPreferences에 `DiaryEntry` JSON 문자열 목록과 사용자 키워드를 저장한다.

## 선택 이유

- Web 데모와 로컬 실행에서 동일하게 사용할 수 있다.
- 서버 없이 오프라인 핵심 흐름을 유지한다.
- 현재 데이터 규모에 충분하고 구현을 안정화하기 쉽다.
- Repository 분리 덕분에 향후 다른 DB로 교체 가능하다.

## 대안

| 대안 | 장점 | 이번 범위에서 제외한 이유 |
|---|---|---|
| Isar | 구조화 데이터와 검색에 적합 | 초기 설정과 스키마 관리가 현재 범위보다 큼 |
| SQLite | 범용성과 쿼리 | 단순한 프로토타입 기록에 과도함 |
| 서버 DB | 여러 기기 동기화 | 오프라인 우선 목표와 발표 안정성에 불리함 |

## 결과와 제한

- 새로고침 후에도 기록과 사용자 키워드가 유지된다.
- 같은 날짜를 저장하면 기존 기록을 교체한다.
- 데이터가 많아지거나 검색 조건이 복잡해지면 Isar 또는 SQLite로 교체한다.
- 저장 실패 시 앱 상태를 유지하고 사용자에게 오류를 알린다.
