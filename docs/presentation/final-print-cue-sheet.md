## 절대 읽지 말고 말할 문장 뼈대

1. “하루톡은 긴 일기를 쓰게 하는 앱이 아니라, 기록을 시작하기 쉽게 만든 앱입니다.”
2. “사용자는 기분, 활동, 만족도만 선택하고 앱은 그 사실만으로 한 줄을 만듭니다.”
3. “구현은 Flutter 화면, Controller 흐름, 규칙 로직, Repository 저장소로 나눴습니다.”
4. “현재 53개 테스트와 GitHub Actions로 기능과 배포를 검증했습니다.”
5. “짧은 기록이 쌓이면 감정잔디와 생활 패턴으로 회고할 수 있습니다.”

## 시간이 부족하면 줄일 부분

- 5페이지 기술 설명은 `Flutter + Repository 구조`까지만 말한다.
- 6페이지 시행착오는 `Drift 저장과 ADR 기록`까지만 말한다.
- 8페이지 기대 효과는 `감정잔디와 생활 패턴` 한 문장으로 줄인다.

## 질문 대비 핵심 답변

- 왜 Flutter? → 모바일 UI와 Web 발표 데모를 한 코드베이스로 만들 수 있어서.
- AI는 실제 API인가? → 현재는 규칙 기반 한 줄 생성이고, 외부 AI API는 향후 확장 후보.
- 저장은 무엇을 쓰나? → Drift + SQLite, Web에서는 WASM 기반 SQLite.
- 구조는? → 화면 → Controller → Rules/Model → Repository → Drift DB.
- AI Agent는 어디까지 썼나? → 초안 생성과 구현 보조에 활용했고, 실행·수정·검증은 직접 했다.
