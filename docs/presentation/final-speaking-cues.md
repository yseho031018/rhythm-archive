# 하루톡 최종 발표 말하기 카드

> 슬라이드의 문장을 읽지 않는다. 각 장에서 아래 키워드만 보고 한 가지 이야기를 전달한다.

## 전체 시간과 페이지 이동

- 본 발표: 4분 30초
- 시연 영상: 30초
- 각 장의 마지막 문장을 끝낸 뒤 오른쪽 방향키로 넘긴다.
- 3분 50초에 9페이지 시연 화면으로 넘어가지 못했다면 앞 설명을 줄인다.

## 1. 표지 · 0:00–0:15

핵심 키워드: `세 번의 선택`, `기록 시작`, `생활 회고`

말할 이야기:

- 하루톡은 긴 일기를 대신 써주는 앱이 아니다.
- 기록을 시작하기 쉽게 만들고 생활을 돌아보게 하는 앱이다.
- 마지막: 앱 이름을 말한 뒤 `→ 다음 페이지`

## 2. 비전과 문제 · 0:15–0:45

핵심 키워드: `귀찮음`, `기록 중단`, `목록에 머묾`

말할 이야기:

- 긴 글을 쓰려다 미뤄본 개인 경험을 한 문장으로 말한다.
- 기록이 쌓여도 어떤 활동이 기분에 영향을 주는지 알기 어렵다고 연결한다.
- 마지막: 문제 두 가지를 연결한 뒤 `→ 다음 페이지`

## 3. 사용자 시나리오 · 0:45–1:15

핵심 키워드: `하나씩 질문`, `기분·활동·만족도`, `한 줄`

말할 이야기:

- 토리가 질문을 한 번에 하나씩 보여주어 입력 부담을 낮춘다.
- 잘못 선택하면 이전 질문으로 돌아가도 선택값이 유지된다.
- 현재 한 줄 생성은 규칙 기반이며 사용자가 다시 생성·수정할 수 있다.
- 마지막: 생활 패턴으로 연결한 뒤 `→ 다음 페이지`

## 4. WBS와 진행 과정 · 1:15–1:40

핵심 키워드: `UX 프로토타입`, `핵심 기능`, `100%`

말할 이야기:

- 사용자 문제와 질문형 UX를 먼저 정리하고 기록·저장 기능을 구현했다.
- 감정잔디·생활 패턴을 연결한 뒤 테스트와 배포 안정화에 집중했다.
- 마지막: 현재 단계 설명 후 `→ 다음 페이지`

## 5. 기술과 앱 구조 · 1:40–2:15

핵심 키워드: `Flutter`, `Controller`, `Rules`, `Repository`

말할 이야기:

- 화면은 표시, Controller는 흐름, Rules는 분석, Repository는 저장을 담당한다.
- Repository 인터페이스 덕분에 테스트 저장소와 실제 저장소를 교체할 수 있다.
- 마지막: Repository의 장점을 말한 뒤 `→ 다음 페이지`

## 6. 시행착오와 ADR · 2:15–2:50

핵심 키워드: `설명 가능성`, `Repository`, `Drift`, `자동 이전`

말할 이야기:

- SharedPreferences로 시작했지만 Repository 경계 덕분에 화면 변경 없이 Drift로 교체했다.
- 기존 기록 자동 이전과 WebAssembly SQLite까지 직접 검증했다.
- 마지막: ADR에 기록했다고 말한 뒤 `→ 다음 페이지`

## 7. 품질과 배포 · 2:50–3:25

핵심 키워드: `50`, `0`, `Web Release`, `GitHub Pages`

말할 이야기:

- Controller·패턴 분석 단위 테스트와 이전 이동·월간/연간 잔디를 포함한 통합 위젯 테스트를 했다.
- JSON 백업 복원도 검증해 로그인 없이 기록을 사용자가 직접 소유하게 했다.
- GitHub Actions가 PR·push에서 생성 코드, 분석, 테스트, Web Release 빌드를 자동 확인한다.
- 빌드는 산출물 생성, 배포는 URL 공개라는 차이를 짧게 설명한다.
- 마지막: GitHub Pages 배포를 말한 뒤 `→ 다음 페이지`

## 8. 활용과 기대 효과 · 3:25–3:50

핵심 키워드: `월간·연간 감정잔디`, `생활 패턴`, `회고`

말할 이야기:

- 한 줄이 누적되면 친구와 함께한 날의 만족도처럼 생활 관계를 보여준다.
- 표본이 부족할 때 관계를 단정하지 않는다.
- 마지막: 장기 회고를 말한 뒤 `→ 다음 페이지`

## 9. 30초 시연 · 3:50–4:20

핵심 키워드: `행복`, `친구`, `5점`, `연간 잔디`, `생활 패턴`

말할 이야기:

- 시작 문장 한 줄만 말하고 영상을 바로 재생한다.
- 영상이 끝나면 지체하지 않고 `→ 다음 페이지`

## 10. AI Agent 활용 · 4:20–4:45

핵심 키워드: `생성`, `직접 검증`, `AGENTS.md`

말할 이야기:

- AI Agent는 기획·코드·테스트·문서 감사에 사용했다.
- 판단과 실행 검증은 직접 했고, 작업 규칙과 암묵지는 AGENTS.md에 통합했다.
- 마지막: AGENTS.md를 말한 뒤 `→ 다음 페이지`

## 11. 마무리 · 4:45–5:00

핵심 키워드: `시작하기 쉽게`, `생활을 돌아보기`

말할 이야기:

- 첫 장의 비전을 다시 한 번만 말하고 끝낸다.
- `감사합니다` 이후에는 페이지를 넘기지 않는다.

## 연습 규칙

1. 첫 두 번은 대본을 보면서 4분 30초를 맞춘다.
2. 다음 세 번은 이 카드만 보고 설명한다.
3. 슬라이드의 문장과 똑같은 문장을 연속으로 읽으면 다시 자기 말로 바꾼다.
4. 시간을 초과하면 기술 이름보다 사용자 문제와 구현 근거를 남긴다.
5. 마지막 연습은 URL 접속부터 타이머, 발표, 영상까지 실제 순서로 진행한다.
