# Rhythm 2분 발표 대본

## 목적

중간 발표에서 `Rhythm` 프로젝트의 문제 정의, 핵심 기능, 기술 선택, 현재 진행 상황, 데모 흐름을 2분 안에 설명하기 위한 대본이다.

## 발표 대본

안녕하세요. 앱 프로그래밍 개인 프로젝트로 **Rhythm**이라는 앱을 준비한 윤세호입니다.

Rhythm은 **감정의 파도를 시각화하는 다이어리 앱**입니다. 기존 다이어리 앱은 매일 길게 기록해야 해서 부담이 있고, 기록이 쌓여도 리스트나 숫자로만 보여주는 경우가 많다고 생각했습니다.

그래서 Rhythm은 **에너지, 감정, 활동을 30초 안에 기록**하고, 그 데이터를 **Wave Graph**로 표현합니다. 사용자는 오늘 탭에서 에너지와 감정 키워드, 활동 태그를 선택하고, 저장 후 파동 그래프가 어떻게 바뀌는지 확인합니다. 이후 히스토리에서 날짜별 기록을 다시 보고, 패턴 탭에서 평균 에너지와 자주 나온 감정, 활동을 확인하는 흐름입니다.

현재는 중간 발표용 Flutter Web 데모로 입력 화면, 감정별 파동 그래프, 히스토리, 패턴 카드까지 구현했습니다.

기술 스택은 **Flutter, Riverpod, Isar, Layered Architecture**로 정했습니다.  
Flutter는 CustomPainter 기반 시각화를 만들기 좋고, 최종 구조에서는 Riverpod, Isar, Layered Architecture를 적용해 화면, 상태, 저장소 역할을 분리할 계획입니다. 특히 Wave Graph를 그리는 부분과 감정 데이터를 계산하는 부분을 분리하는 것을 핵심 원칙으로 잡았습니다.

사용자 시나리오는 오늘의 리듬을 남기고, 저장된 기록을 다시 확인한 뒤, 패턴을 통해 생활 리듬을 돌아보는 흐름으로 구성했습니다. GitHub Pages에서는 발표자료와 WBS/Gantt 페이지, Flutter Web 데모를 바로 확인할 수 있게 준비했습니다.

기말까지는 로컬 저장 구조를 안정화하고, 감정-활동 상관 분석과 주간 리듬 리포트를 추가해서 단순 기록 앱이 아니라 생활 리듬을 분석하는 앱으로 발전시키겠습니다. 추가 후보로는 월간 캘린더 뷰와 날씨 API 연동도 검토하고 있습니다.

감사합니다. 질문 받겠습니다.
